H7 TOF Safety Console

这不是一页单纯的串口监视器，而是一张给安全冗余板做联调的中控台。页面会直接读取 STM32 的串口状态帧， 实时显示 4 路 TOF 的距离、当前参与保护的方向逻辑、急停来源、掩码变化和原始串口日志。

安全窗口 363 mm ~ 419 mm 兼容旧版与扩展版 H7TOF 支持逐行 JSON 调试输入
USB CDC 链路 未连接
急停提示灯 待机

板端主动急停和外部急停都会让这里亮红，用于第一时间确认是安全链路拦下来的，还是外部线触发的。

联调提示

建议使用 Edge 或 Chrome，并通过 http://localhost 打开。网页和 ROS SDK 不能同时占用同一个串口。

Protocol

协议与兼容说明

  • 推荐状态帧：$H7TOF,seq,tof1,tof2,tof3,tof4,estop,self_estop,ext_estop,active_mask,trip_mask,valid_mask,fault_mask,motion_mode,takeover*CS
  • 兼容旧版联调帧：$H7TOF,seq,tof1,tof2,tof3,tof4,estop,valid_mask,fault_mask*CS
  • 页面按行解析，支持 \r\n，也兼容单行 JSON。
  • 无效距离建议发 65535，页面会显示为无效。
  • 若开启 ROS SDK，请先关闭浏览器对该串口的占用。
Strategy Snapshot

方向策略速览

这是页面固定展示的判定规则，便于现场联调时快速对照是否与板端策略一致。

前进
只看前侧

启用 TOF1 + TOF4，后方通道继续上报，但不再参与急停触发。

后退
只看后侧

启用 TOF2 + TOF3，前方通道只做观测，不参与急停决策。

转向 / 混合运动
四路全保护

只要存在转向、横移或混合运动，页面就应看到 active_mask = 0x0F。

失联保护
回退到最保守

主机控制超时后按四路全保护处理，防止通信中断时出现安全盲区。
